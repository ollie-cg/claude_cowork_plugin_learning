# Plugin Brands Weekly Client Update Email — v1

Sibling of `weekly-client-update-deck`. Same data sourcing and data rules — different output channel.

## Input (required)

**Pipeline MD file** — `docs/temporary/{brand-slug}-pipeline-{YYYY-MM-DD}.md`, generated by the `brand-pipeline-report` skill.

## Personalisation (ask up to 2; skip if provided)

1. Rep name — defaults to the most common `Owner` across the MD's in-play deals.
2. Greeting — defaults to `Hi [Client],`. The user fills in the bracketed name before sending.

## Procedure

### 1. Parse the MD

- Read `## Summary` table → stage counts + total.
- Collect in-play deals (Proposal Feedback Pending → Proposal → Feedback Received → Feedback Pending → Follow Up → Discovery) with buyer name, sites, route to market, short state from the MD's per-deal `Summary` line.
- Collect Won deals with buyer name, sites, route to market.

### 2. Compose the body — 4 sections, no comparisons

**Section A — One-line lead.** "Quick update on the {BRAND} buyer pipeline this week."

**Section B — Pipeline shape.** An HTML table with all 9 stages + Total. Counts come from the MD summary table verbatim.

**Section C — Key highlights (3–4 bullets).** Each bullet is factual, uses named buyers, and never compares deals or brands.
- Bullet 1 — the shape: where the weight sits (e.g. "fully qualified", "large top-of-funnel", "middle-heavy"). Describe without ranking.
- Bullet 2 — the late-stage deals (Proposal Feedback Pending + Proposal). Name 2-4 buyers with the most advanced stages.
- Bullet 3 — mid-stage momentum (Feedback Received / Feedback Pending) if any; otherwise drop this bullet.
- Bullet 4 — wins secured this period. Name every Won deal (not just the biggest).

**Section D — Actions next week (3–4 bullets).** Derived from stage mechanics:
- Convert Proposal Feedback Pending → listing.
- Progress Proposal → active feedback.
- Move Feedback Pending / Follow Up forward.
- If Discovery is empty, flag "rebuild top-of-funnel volume".

Close: `Any questions or anything you want us to push harder on, let me know.` + sign-off (`Best,` / rep name / `Plugin Brands.`).

### 3. Render two artefacts

Save both to the project folder (or a path the user specifies):

1. **`{brand-slug}-email-{YYYY-MM-DD}.md`** — Markdown source. Edit-friendly, archive-friendly.
2. **`{brand-slug}-email-{YYYY-MM-DD}.html`** — HTML version. Structure:
   - `<body>` with `font-family: -apple-system, Helvetica, Arial, sans-serif; font-size: 14px; color: #111; line-height: 1.5;`
   - `<table>` with `border-collapse: collapse; border: 1px solid #ccc;`, cell padding 6px, header row `background: #f4f4f4;`, every cell has `border: 1px solid #ccc;` so Gmail's paste handler preserves borders.
   - `<ul>` for bullets.
   - `<b>` for emphasis (don't use `<strong>` — some paste handlers drop it).
   - No external images, no CSS classes, no `<style>` block. Inline styles only.

### 4. Push to clipboard (macOS default)

```
textutil -stdin -format html -convert rtf -stdout < {html_path} | pbcopy
```

This puts RTF on the clipboard. Gmail's paste handler converts RTF back to rich HTML on paste — table, bullets and bold survive.

**Fallback for non-macOS:** skip the pbcopy step and tell the user to drag the `.html` file into Gmail (Gmail imports it as a formatted draft).

### 5. Return to the user

- **Subject line** (suggested): `{BRAND} — Weekly Pipeline Update (week ending {DD MMMM})`
- **File paths** (both `.md` and `.html`).
- **Confirmation:** "Rich-text email on your clipboard — paste into a new Gmail compose with ⌘V."

## Data rules (same as deck skill)

- Only MD data + user-provided rep/greeting. Do not invent meetings, channels, samples, outreach types.
- No cross-brand or cross-deal comparisons. No ranking language ("top performer", "biggest win", "best deal").
- Named buyers only — never "a major buyer" or "a key account".
- If the MD's summary says `no {BRAND}-specific mention appears in the fetched engagements`, describe the deal by its stage mechanics only — don't invent buyer intent.
- Stale deals (latest touch is an auto-reply / OOO) should be flagged honestly if they appear in highlights.
- Every number in the email must match the MD's summary table or per-deal counts — no arithmetic that re-aggregates stages.

## Output format (to user)

```
Subject: {brand} — Weekly Pipeline Update (week ending {date})

Rich-text email copied to clipboard. Paste with ⌘V into Gmail.

Saved:
- {brand-slug}-email-{date}.md
- {brand-slug}-email-{date}.html

Defaulted:
- Rep: {rep name} (most common owner in in-play deals)
```

Nothing else. No commentary, no alternatives, no preview of the email text in the chat — it's on the clipboard.
